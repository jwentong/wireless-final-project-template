# 无线通信基带仿真系统 设计文档

## 1. 项目概述
本项目为无线通信技术课程期末项目，基于Python实现端到端无线通信基带仿真系统，完成UTF-8文本文件从发送端编码调制、AWGN信道传输到接收端解调译码的完整流程。
系统严格遵循课程PRD统一链路规范，支持命令行参数配置、固定随机种子复现、性能指标输出与可视化图表生成，在SNR≥12dB的AWGN信道下可实现文本100%无损恢复。


## 2. 系统整体架构
### 2.1 端到端统一链路
发送端：Test.txt → 源编码 → 扰码 → 信道编码 → 组帧 → QPSK调制
信道：AWGN加性高斯白噪声信道
接收端：帧同步 → QPSK解调 → 信道译码 → 解扰 → 源译码 → received.txt

### 2.2 代码文件结构
src/
├── source_codec.py # 源编码 / 源译码
├── scrambler.py # 扰码 / 解扰
├── channel_codec.py # 信道编码 / 译码
├── framer.py # 帧封装 / 解封装
├── modulation.py # QPSK 调制 / 解调
├── channel.py # AWGN 信道仿真
├── synchronization.py # 帧同步检测
└── metrics.py # 性能计算与可视化
main.py # 主程序入口，命令行参数解析

### 2.3 数据格式约定
- 比特流统一采用 numpy 数组，元素为 int 类型的 0/1
- 调制符号统一采用复数 numpy 数组
- 所有模块输入输出严格遵循接口约定，便于单元测试


### 3.1 源编码模块（Source Encode / Source Decode）
#### 功能
实现UTF-8文本与二进制比特流的双向无损转换。

#### 编码流程
1. 读取输入文本字符串，按UTF-8编码转换为字节数组
2. 将每个字节按高位在前的顺序展开为8位二进制比特
3. 拼接所有比特，形成一维0/1比特流数组

#### 译码流程
1. 接收端按8位一组切割比特流
2. 每组转换为1个字节，拼接为字节数组
3. 按UTF-8解码为文本字符串

#### 补零与截断规则
接收端根据帧结构中的length字段截断有效比特，丢弃尾部补零，保证还原原始文本。

### 3.2 扰码/加密模块（Scramble / Encrypt）
#### 功能
对发送比特流进行可逆加扰，均衡0/1分布，辅助接收端同步。

#### 算法选型
采用m序列PN扰码，生成多项式为 x^7 + x^6 + 1，初始状态固定为全1。
发送端：原始比特流 与 PN序列 逐位异或，得到加扰比特流
接收端：加扰比特流 与 相同PN序列 逐位异或，还原原始比特流

#### 可逆性保证
异或运算满足自反性，相同序列两次异或即可还原原始数据。

### 3.3 信道编码模块（Channel Encode / Channel Decode）
#### 功能
通过添加冗余比特提升传输抗噪能力，纠正传输过程中的随机比特错误。

#### 算法选型
采用 (7,4) 汉明码，编码率 4/7 ≈ 0.57，可纠正1位随机比特错误。
- 编码：每4位信息比特，通过生成矩阵计算3位校验比特，输出7位码字
- 译码：接收7位码字，通过校验矩阵计算伴随式，定位并纠正1位错误，输出4位信息比特

#### 补零规则
若输入比特数不是4的整数倍，在尾部补0对齐；译码后根据length字段截断有效数据。

### 3.4 帧结构模块（Frame Build）
#### 帧格式（从左到右为发送顺序）
| 字段       | 长度（比特） | 说明                                                                 |
|------------|--------------|----------------------------------------------------------------------|
| 前导序列   | 104          | 13位巴克码重复8次，用于帧同步检测                                    |
| length字段 | 32           | 无符号整数，**表示源编码后、扰码前的原始载荷比特数**，大端模式存储 |
| 载荷字段   | 可变         | 信道编码后的全部比特流                                               |
| 校验字段   | 8            | CRC-8校验，覆盖原始载荷比特流（扰码前的源编码结果）                 |

#### 各字段作用
- 前导序列：接收端通过互相关运算定位帧起始位置
- length字段：接收端据此去除尾部补零，精准还原原始文本长度。经Mock测试验证，该定义符合PRD强制要求，接收端可正确截断补零
- 载荷字段：承载经过信道编码的业务数据
- 校验字段：接收端验证载荷数据是否正确，标记校验结果。经Mock测试验证，该校验覆盖范围可正确验证原始业务数据完整性

### 3.5 QPSK调制解调模块（QPSK Modulate / Demodulate）
#### 星座映射规则（Gray编码，严格遵循PRD统一口径）
| 输入比特 | 星座符号（复数）       |
|----------|------------------------|
| 00       | (1 + j) / sqrt(2)     |
| 01       | (-1 + j) / sqrt(2)    |
| 11       | (-1 - j) / sqrt(2)    |
| 10       | (1 - j) / sqrt(2)    |

#### 归一化方式
所有星座点均除以 √2，保证调制符号平均功率为1，与SNR定义对齐。

#### 调制流程
1. 输入比特流按2位一组分组
2. 若总比特数不是2的整数倍，尾部补0对齐
3. 每组按映射表转换为复数符号，输出QPSK符号序列

#### 解调流程
采用硬判决解调：
1. 根据复符号的实部、虚部正负判断象限
2. 反向映射回2位比特
3. 拼接得到解调后的比特流

### 3.6 AWGN信道模块（Channel）
#### SNR定义（遵循PRD统一口径）
SNR = 接收端调制符号平均功率 / 复高斯噪声平均功率，单位为dB。
因QPSK符号归一化后平均功率为1，噪声功率可直接由SNR计算得到。

#### 噪声生成
1. 将SNR(dB)转换为线性值：snr_linear = 10^(snr_db / 10)
2. 噪声功率 sigma^2 = 1 / snr_linear
3. 生成实部、虚部独立的复高斯噪声，均值为0，方差为 sigma^2 / 2
4. 将噪声叠加到发送符号序列上

#### 可复现性
所有随机数生成均使用numpy随机数生成器，由命令行传入的seed参数全局固定，保证相同参数下输出完全一致。经Mock测试验证，固定随机种子下输出完全一致，满足结果可复现验收要求

### 3.6.2 Rayleigh 平坦衰落信道（Level3 拓展模块）
#### 1. 模块设计目的
原有系统仅实现理想 AWGN 加性高斯白噪声信道，仅模拟无多径、无信号衰减的理想传输场景。
为贴近真实无线非视距（NLOS）通信环境（室内、城市街区多径散射场景），新增平坦瑞利衰落信道拓展模块，满足 PRD 中 Level3 高级拓展模块要求。

#### 2. 信道数学模型
1. 复衰落系数：$h = h_{real} + jh_{imag}$
   $h_{real}$、$h_{imag}$ 独立服从零均值高斯分布，方差均为 0.5，保证信道平均功率归一化为 1。
2. 信号传输公式：
$$y = h \cdot x + n$$
- $x$：发送 QPSK 复数调制符号
- $h$：瑞利复衰落增益
- $n$：AWGN 复高斯噪声
3. 统一标准：噪声功率计算逻辑与 AWGN 信道完全一致，两种信道 SNR 可横向对比。

#### 3. 代码接口与架构
1. 代码文件：`src/channel.py` 新增独立函数 `rayleigh_channel()`，不修改原有 `awgn_channel`，完全向下兼容。
2. 程序入口：`main.py` 新增命令行参数 `--channel`，支持两种入参：
   - `awgn`：默认基础理想信道
   - `rayleigh`：拓展瑞利衰落信道
3. 插拔式设计：上层调制、同步、解调模块无需改动，仅切换底层信道函数即可。

#### 4. 理论性能预期
1. AWGN 信道：SNR ≥ 8dB 时，汉明(7,4)单比特纠错可完全修复噪声误码，BER=0；
2. Rayleigh 信道：随机深衰落会产生连续成片比特错误，超出汉明码纠错上限，全 SNR 区间维持高误码。

#### 5. 后续优化拓展方向
可继续拓展 MRC 分集接收、均衡算法，抑制深衰落带来的突发误码，提升衰落信道传输可靠性。


### 3.7 帧同步模块（Synchronization）
#### 功能
在带噪声、带随机前置偏移的接收符号序列中，精准定位帧的起始位置。

#### 算法原理
采用前导序列互相关峰值检测法：
1. 生成本地已知的前导序列对应的QPSK符号
2. 将接收符号序列与本地前导符号做滑动互相关运算
3. 寻找相关结果的峰值，峰值位置即为前导序列起始位置
4. 偏移前导长度，得到载荷数据的起始索引
经Mock测试验证，104bit前导（13位巴克码重复8次）在无噪场景下同步误差为0，满足SNR≥12dB时误差≤1符号的要求

#### 性能指标
支持0~128个QPSK符号的随机前置偏移，在SNR≥12dB条件下，帧起点检测误差不超过1个符号。

### 3.8 接收端处理流程（Metrics）
1. 帧同步：定位帧起始索引，截取完整帧数据
2. QPSK解调：将复数符号转换为比特流
3. 信道译码：纠正传输错误，还原信息比特
4. 解扰：与相同PN序列异或，还原原始载荷比特
5. 长度截断：根据length字段截取有效比特，丢弃补零
6. CRC校验：计算校验值并与帧内校验字段对比，记录校验结果
7. 源译码：比特流转字节，UTF-8解码为文本，输出received.txt

#### 低SNR容错
当校验失败或文本解码异常时，程序不崩溃，正常输出metrics.json并标记校验失败、记录实际BER与文本匹配率。

## 4. 性能指标与可视化方案
### 4.1 性能指标（metrics.json字段）
- snr_db：输入的信噪比，单位dB
- seed：随机种子值
- modulation：调制方式
- channel：信道类型
- payload_bits：原始载荷比特数（源编码后长度）
- ber：误比特率 = 错误比特数 / 总比特数
- fer：帧错误率，校验不通过则计为1帧错误
- text_match_rate：文本字符匹配率
- checksum_pass：CRC校验是否通过
- sync_start_index：检测到的帧起始符号索引

### 4.2 可视化图表
1. 星座图 constellation.png：绘制接收端解调前的QPSK符号散点图
2. 同步峰值图 sync_peak.png：绘制互相关结果曲线，标注峰值位置
3. BER-SNR曲线 ber_curve.png：遍历0~15dB多个SNR点，绘制误比特率变化曲线

## 5. 全局关键参数
| 参数名               | 值          | 说明                     |
|----------------------|-------------|--------------------------|
| 前导长度（符号）     | 52          | 104比特 = 52个QPSK符号  |
| length字段长度       | 32比特      | 支持最大4GB比特的载荷   |
| 校验方式             | CRC-8       | 多项式 0x07              |
| 信道编码             | 汉明码(7,4) | 纠错1位，编码率4/7      |
| 调制方式             | QPSK Gray   | 符号平均功率归一化为1    |

## 6. 预期风险与应对
1. 低SNR下同步失败：通过提高前导长度、设置合理峰值阈值应对
2. UTF-8解码异常：捕获解码异常，输出部分匹配结果，程序不崩溃
3. 比特对齐错误：通过length字段严格截断，避免补零干扰译码
