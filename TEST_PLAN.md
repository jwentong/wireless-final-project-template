# 无线通信基带仿真系统 测试计划

## 1. 测试目标
1. 验证 DESIGN.md 中各模块设计的正确性与接口可行性
2. 覆盖课程PRD规定的全部公开测试场景，确保基础功能达标
3. 提前发现设计缺陷，在编码前完成方案修订，降低后期返工成本
4. 为后续单元测试、集成测试和端到端验收提供明确的判定标准

## 2. 测试环境
- 开发语言：Python 3.9+
- 依赖库：numpy、matplotlib
- 测试方式：Mock 极简脚本验证 + 后续 pytest 单元测试
- 基准输入：项目自带 Test.txt（UTF-8 编码）

## 6. 测试通过判定标准
### 6.1 公开测试通过标准
1. 全部单元测试用例通过
2. SNR≥12dB、AWGN、固定seed条件下，端到端文本完全一致
3. metrics.json字段完整、数值正确
4. 至少生成2张可视化图表
5. 异常场景下程序不崩溃，有合理错误提示

### 6.2 Mock测试目标
本次Mock阶段不要求完整代码，仅通过极简脚本验证核心接口和设计逻辑的可行性，发现设计缺陷并修订DESIGN.md。
