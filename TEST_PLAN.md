# Test Plan

## 1. Test Scope

The test plan validates the required wireless chain from `Test.txt` to `results/received.txt` and checks module-level behavior for hidden-test robustness.

## 2. Public Test Mapping

| Requirement | Test Method |
|---|---|
| Required files exist | `pytest public_tests/test_01_structure_and_documents.py` |
| Source Encode / Source Decode reversible | `tests/test_wireless_pipeline.py::test_utf8_source_codec_round_trips_chinese_text` and public tests |
| Frame contains preamble, length, payload, checksum | `test_frame_round_trip_preserves_odd_length_payload` and public tests |
| Scramble / Encrypt reversible | public tests call `src.crypto.scramble` and `descramble` |
| Channel Encode / Decode reversible | public tests call `src.channel_coding.channel_encode` and `channel_decode` |
| QPSK Gray mapping | `test_qpsk_uses_required_gray_mapping_and_demaps_without_noise` |
| AWGN fixed seed reproducible | `test_awgn_is_seed_reproducible` |
| Synchronization detects random prefix | `test_detect_frame_start_handles_symbol_offset` and public tests |
| Unified CLI | `test_cli_generates_metrics_and_received_text` and public tests |
| Results and plots | CLI plus public tests for `metrics.json` and at least two PNG files |
| Level 3 Rayleigh extension | `test_rayleigh_flat_channel_can_be_equalized_with_known_gain` and `test_cli_supports_rayleigh_extension_with_equalization` |
| Web simulation interface | `tests/test_web_app.py` verifies the page, `/api/run`, full `stage_trace`, module options, and empty-input validation |
| Extension experiments | `tests/test_experiments.py` verifies SNR/coding/channel experiment outputs |

## 3. Mock Tests

1. Source codec mock: encode a Chinese UTF-8 sentence, decode it, and require exact equality.
2. Frame mock: build a frame for an odd-length payload and parse it back to verify length and checksum.
3. QPSK mock: modulate the four PRD bit pairs and verify quadrants and unit-power normalization.
4. Synchronization mock: insert a 25- or 37-symbol prefix before the preamble and verify detected start.
5. End-to-end mock: run `run_pipeline()` at 12 dB and require exact text recovery.

## 4. Commands

Install dependencies:

```bash
pip install -r requirements.txt
```

Run local tests:

```bash
pytest tests -q
```

Run public tests:

```bash
pytest public_tests -q
```

Run the required CLI:

```bash
python main.py --input Test.txt --output results/received.txt --snr 12 --seed 2026 --mod qpsk --channel awgn
```

Run the Level 3 extension:

```bash
python main.py --input Test.txt --output results/received.txt --snr 18 --seed 2026 --mod qpsk --channel rayleigh
```

Run the Web interface:

```bash
python web_app.py --host 127.0.0.1 --port 8000
```

Run the extension experiments:

```bash
python scripts/run_experiments.py --input Test.txt --output-dir results/experiments --snrs 0,3,6,9,12,15 --seed 2026
```

## 5. Hidden-Test Considerations

Hidden validation may use different text, SNR, seed, and prefix offset. The code avoids hardcoded input and computes all lengths from runtime data. For low SNR, exact text recovery is not guaranteed, but the system should still exit normally and produce BER, FER, text_match_rate, checksum_pass, and failure_reason. The Rayleigh extension is tested separately so it does not weaken the required AWGN baseline.

## Additional Level 3 Advanced Tests

Advanced modules are covered by `tests/test_advanced_modules.py`:

- Adaptive modulation thresholds and round trips for BPSK/QPSK/16-QAM.
- Convolutional encoder plus Viterbi decoder with a corrected coded-bit error.
- OFDM modulation/demodulation round trip.
- MRC2 diversity shape and BER sanity check.
- End-to-end all-advanced pipeline.
- CLI all-advanced flag coverage.

Experiment evidence is generated by `scripts/run_experiments.py`, including the `advanced-all` scenario.
