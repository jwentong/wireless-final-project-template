# MOCK_TEST_REPORT.md — Mock 测试报告

## 概述

本文档记录在正式编码前对关键模块接口进行的 mock 测试结果。测试目标：验证 DESIGN.md 中定义的接口、帧结构、同步流程和端到端流程是否可行，并根据测试结果修订设计。

---

## Mock 测试记录

### Mock-1: 源编码 UTF-8 往返

- **测试目标**：验证 UTF-8 文本 → bitstream → UTF-8 文本的可逆性
- **测试数据**：`"无线通信技术课程要求学生理解调制编码信道和接收机处理"`
- **测试方法**：调用 `source_encode()` 获取比特流，再调用 `source_decode()` 恢复文本
- **测试结果**：✅ 通过
  - bitstream 长度为 360 bit（45 字节 × 8），可被 8 整除
  - 恢复文本与原文完全一致
- **发现问题**：无

### Mock-2: 帧结构字典格式

- **测试目标**：验证 `build_frame()` 返回的 dict 结构包含 PRD 要求的四个字段
- **测试数据**：2400 bit 随机 payload
- **测试方法**：调用 `build_frame(payload)`，检查返回 dict 的键
- **测试结果**：✅ 通过
  - 返回 dict 包含 `preamble`（32 bit）、`length`（2400）、`payload`（2400 bit）、`crc`（16 bit）
- **发现问题**：无

### Mock-3: 帧封装解析往返（奇数长度 payload）

- **测试目标**：验证 257 bit 奇数长度 payload 能正确封装和解析
- **测试数据**：257 bit 随机 payload（奇数长度）
- **测试方法**：`build_frame()` → 序列化 → `parse_frame()`
- **测试结果**：✅ 通过
  - 解析得到的 payload 与输入一致（前 257 bit）
  - `length` 字段值为 257
  - padding bit 被正确去除
- **发现问题**：无

### Mock-4: 扰码往返测试

- **测试目标**：验证 XOR PN 扰码的可逆性
- **测试数据**：511 bit 随机比特，seed=2026
- **测试方法**：`scramble(bits, seed=2026)` → `descramble(scrambled, seed=2026)`
- **测试结果**：✅ 通过，完全恢复
- **发现问题**：无

### Mock-5: 汉明(7,4) 信道编码往返（无噪）

- **测试目标**：验证编码后译码能完全恢复原始数据
- **测试数据**：400 bit 随机比特
- **测试方法**：`channel_encode(bits)` → `channel_decode(coded)`
- **测试结果**：✅ 通过
  - 原始 400 bit 分 100 组，每组 4 bit → 7 bit，共 700 bit
  - 译码后完全恢复
- **发现问题**：输入长度需为 4 的倍数；末尾不足 4 bit 需补零。已在设计中明确。

### Mock-6: QPSK 星座象限验证

- **测试目标**：验证 Gray 编码星座映射
- **测试数据**：`[0,0,0,1,1,1,1,0]`（比特对 00, 01, 11, 10）
- **测试方法**：`qpsk_modulate()` → 检查每个符号的 I/Q 符号
- **测试结果**：✅ 通过
  - 00 → 第一象限 (I>0, Q>0)
  - 01 → 第二象限 (I<0, Q>0)
  - 11 → 第三象限 (I<0, Q<0)
  - 10 → 第四象限 (I>0, Q<0)
  - 平均功率 = 1.0（归一化因子 1/√2 正确）
- **发现问题**：无

### Mock-7: QPSK 无噪往返

- **测试目标**：验证调制+解调在无噪声下无误码
- **测试数据**：512 bit 随机比特
- **测试方法**：`qpsk_modulate()` → `qpsk_demodulate()`
- **测试结果**：✅ 通过，BER = 0
- **发现问题**：需确保输入 bit 数为偶数。帧封装层的 padding 负责保证此条件。

### Mock-8: AWGN 信道可复现性

- **测试目标**：验证固定 seed 下 AWGN 输出可复现
- **测试数据**：4 个 QPSK 符号（归一化功率），SNR=12 dB, seed=2026
- **测试方法**：连续两次调用 `awgn()`
- **测试结果**：✅ 通过，两次输出完全一致（`np.allclose` 容差内）
- **发现问题**：需使用独立的 NumPy RNG (`np.random.default_rng(seed)`)，避免全局状态污染。

### Mock-9: 同步检测偏移

- **测试目标**：验证 preamble 互相关能检测帧起点
- **测试数据**：25 个噪声符号 + preamble（16 QPSK 符号）+ 80 个载荷符号
- **测试方法**：`synchronize(received, preamble)`
- **测试结果**：✅ 通过
  - 检测到的起点 = 25（误差 0，在 ±1 容差内）
- **发现问题**：同步在低 SNR（< 5 dB）下可能失败。设计上需要确保 SNR ≥ 8 dB 时可靠同步。

## 设计缺陷与修订记录

### 发现的问题 1：QPSK padding 对齐
- **缺陷描述**：初始设计中未明确奇数帧位数的处理方式。
- **修订内容**：在 `build_frame()` 中增加自动检测：若帧位数（64 + N）为奇数，末尾补 1 bit 零。接收端 `parse_frame()` 根据 length 字段精确截取 payload，丢弃 padding。
- **影响模块**：`framing.py`

### 发现的问题 2：信道编码输入对齐
- **缺陷描述**：汉明(7,4) 要求每组 4 bit，但 payload 总 bit 数不一定是 4 的倍数。
- **修订内容**：在 `channel_encode()` 中末尾补零到 4 的倍数；在 `channel_decode()` 中根据原始长度截取。
- **影响模块**：`channel_coding.py`

### 发现的问题 3：AWGN 随机数隔离
- **缺陷描述**：使用全局 `np.random.seed()` 可能被其他模块干扰。
- **修订内容**：统一使用 `np.random.default_rng(seed)` 创建独立 RNG 实例，确保复现性。
- **影响模块**：`channel.py`, `crypto.py`

### 修订后的 DESIGN.md 更新
- 帧结构增加 QPSK 适配 padding 说明
- 信道编码增加补零对齐说明
- AWGN 模块增加独立 RNG 说明
