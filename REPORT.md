# REPORT.md - 实验分析报告

## 1. 测试配置

标准命令：

```bash
python main.py --input Test.txt --output results/received.txt --snr 12 --seed 2026 --mod qpsk --channel awgn
```

Test.txt 为教师提供的约 300 字中文课程描述（UTF-8 编码，源编码后 6128 bit）。

## 2. SNR=12dB 标准结果（公开基础通过条件）

| 指标 | 数值 |
|---|---|
| text_match_rate | 1.0000（received.txt 与 Test.txt 完全一致） |
| ber | 0.000054（信道编码后比特级，FEC 纠错前） |
| fer | 1.0（帧内逐比特级别存在个别噪声翻转，但均被 FEC 纠正） |
| checksum_pass | True（FEC 纠错后端到端 CRC16 校验通过） |
| sync_start_index | 25（与人为插入的同步偏移完全一致） |

结论：在 SNR>=12dB、AWGN、固定 seed=2026 条件下，系统满足 PRD 6.1 节"公开基础通过条件"，文本 100% 恢复。

## 3. QPSK 星座图分析（`constellation.png`）

星座图取自解调前（含 AWGN 噪声）的接收符号。四个理论星座点为 Gray 编码的 QPSK 映射：`00->(1+j)/√2, 01->(-1+j)/√2, 11->(-1-j)/√2, 10->(1-j)/√2`，对应四个象限。在 SNR=12dB 下，噪声云团紧密聚集在四个理论点周围，象限之间没有明显重叠，这也解释了为什么最终比特级误码率能压到 5.4e-5 这么低——判决边界（I=0 和 Q=0 两条轴）几乎不会被噪声跨越。

## 4. BER-SNR 曲线分析（`ber_curve.png`）

曲线扫描了 -2dB 到 16dB 的 SNR 范围，纵轴为对数坐标的 BER。可以看到：
- SNR 较高（>=10dB）时，BER 迅速下降到 1e-4 量级以下，与第 2 节测得的 5.4e-5 数量级一致；
- SNR 较低（<4dB）时，BER 快速上升，重复码（rate=1/3，每 3 位纠 1 位）的纠错能力被突破，误码率迅速逼近未编码 QPSK 的理论水平。

## 5. 同步相关峰值图分析（`sync_peak.png`）

图中横轴为符号位置，纵轴为归一化相关值。可以看到一个清晰的尖峰出现在第 25 个符号位置附近，与代码里人为插入的 25 个偏移符号完全吻合，验证了基于滑动窗口归一化相关的同步算法在 SNR=12dB 下能够精确定位帧起点（PRD 要求误差 <=1 符号，本次实测误差为 0）。

## 6. 低 SNR 下的失败分析（SNR=0dB 对比实验）

同样的 Test.txt，仅将 `--snr` 改为 0，实测结果：

| 指标 | SNR=12dB | SNR=0dB |
|---|---|---|
| text_match_rate | 1.0000 | 0.1429 |
| ber | 0.000054 | 0.163533 |
| checksum_pass | True | False |

**失败原因**：0dB 时噪声功率与信号功率相当，QPSK 判决边界被大量跨越，原始比特误码率大幅上升；虽然重复码理论上能纠正"每 3 位中错 1 位"，但在如此高的原始误码率下，同一分组内 2 位或 3 位同时出错的概率显著上升，多数判决失效，进而导致：(1) 信道解码后比特错误率仍然很高（0.1635）；(2) 帧内 length/checksum 字段本身也可能被噪声破坏，进一步引发帧解析错位；(3) 最终 UTF-8 解码大量字节非法，只能以 `errors="replace"` 兜底，得到 `text_match_rate=0.14` 的严重失真文本。

这说明本系统在低 SNR 下**最先失效的模块是信道编码（FEC）**——重复码的纠错增益有限，是本设计的主要瓶颈，也是 Level 3 提高项中建议用卷积码+Viterbi 译码替换重复码的原因（可以在同样开销下获得更好的编码增益）。系统在该场景下没有崩溃，而是按 PRD 要求正常输出了 BER/FER/text_match_rate 和 `checksum_pass=false` 作为失败标记，满足"低 SNR 行为"这一验收要求。

## 7. 总结

- 基础验收条件（SNR=12dB）100% 达标；
- 系统在低 SNR 下能优雅退化，不崩溃，并给出可解释的失败原因（FEC 纠错能力不足）；
- 同步、调制解调、帧结构、扰码、信道编码各模块均通过了对应的公开测试与 mock 测试；
- 主要限制在于重复码的编码效率和纠错能力偏弱，是后续 Level 3 提高的重点方向。
